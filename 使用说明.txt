  XMU 课程网待办爬取工具 — 使用说明
═══════════════════════════════════════════════

【首次设置】

  python main.py --save-cred      保存学号/密码（base64 存在 data/credentials.json）
  python main.py --run            首次运行即自动登录
  .\schedule.ps1                  注册 Windows 定时任务（每小时后台自动运行）


【日常使用】

  python main.py --run            手动运行一次（爬取 → 生成报告 → 更新桌面图标）
  .\unschedule.ps1                停止定时任务


【账号管理】

  python main.py --save-cred      保存/覆盖当前账号
  python main.py --account 账号名  切换到指定账号（配合 --save-cred / --run 使用）
  python main.py --list-accounts  列出所有已保存账号


【功能说明】

  待办爬取    遍历本学期每门课程，通过 API 获取所有活动及完成度
              自动筛选 completeness < 100 的未完成项目，按截止时间排序
  HTML 报告   生成 output/todos.html（含完成度列、超时分级着色、响应式布局）
  JSON 数据   生成 output/todos.json（含 overdue、completeness、overdue_days 等字段）
  图标提醒    有 24h 内截止且未过期 → 桌面 todos.lnk 变 warning.ico 警示图标
              无紧急待办 → 恢复 normal.ico 正常图标
  超时分级    今天截止 / 超时1天 → 黄色  |  2-7天 → 红色  |  7天以上 → 紫色
  自动登录    Session 过期时用保存的凭据自动填表 XMU CAS 认证
  Cookie 保鲜  每次成功爬取后刷新登录态，避免频繁过期
  后台静默    pythonw.exe 运行，不弹终端窗口


【自动更新】

  开启：在 PowerShell 中执行 .\schedule.ps1
        需要先设置执行策略（仅首次）：
          Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

  关闭：在 PowerShell 中执行 .\unschedule.ps1

  管理：运行 taskschd.msc → 任务计划程序库 → XMU-TronClass-Scraper
        可手动调整执行频率（默认每小时一次）
        可右键 → 运行 立即执行一次


【文件说明】

  data/auth_state.json    浏览器 cookie 等登录态（Playwright 格式）
  data/credentials.json   账号密码（base64 编码，非明文）
  data/normal.ico         正常状态桌面图标
  data/warning.ico        警示状态桌面图标（有超时待办时切换至此）
  data/overdue.ico        保留备用
  output/todos.html       待办报告网页
  output/todos.json       待办数据 JSON


【自动登录说明】

  密码以 base64 存储在本地 data/credentials.json，仅用于自动填表登录 XMU 统一认证。
  首次 --save-cred 后每次 --run 自动登录，无需额外操作。
